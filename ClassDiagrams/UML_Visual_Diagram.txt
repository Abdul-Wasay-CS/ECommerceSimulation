                                          LEGEND & RELATIONSHIP TYPES


  ═══════════════════════════════════════════════════════════════════════════════════════════════════════

  ◇ (Empty Diamond) = Aggregation (Weak Association)
     • "has-a" relationship with sharing
     • Child can exist independently
     • Example: Cart has CartItems (items can be reused)

  ◆ (Filled Diamond) = Composition (Strong Association)
     • "has-a" relationship with ownership
     • Child depends on parent for existence
     • Example: User has Address (address only exists for that user)

  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  △ = Inheritance (Extends/Implements)
     • Child "is-a" Parent
     • Examples:
       - Customer extends User
       - Admin extends User
       - Order extends Cart
       - Clothes extends Product
       - Grocery extends Product
       - Electronics extends Product

  ═══════════════════════════════════════════════════════════════════════════════════════════════════════
