# How to Turn the E-Commerce Project into a Strong OOP Project

Instead of building a **simple store**, design a **system with roles, product types, and services**.

### Core Class Structure

```
pakMart.pakMart.User
 ├── Customer
 └── pakMart.pakMart.Admin

Product
 ├── Electronics
 ├── Clothing
 └── Grocery
```

Additional classes:

```
Cart
CartItem
Order
Payment
StoreManager
InventoryManager
```

Now the system becomes a **real object-oriented design problem**.

---

# Features That Increase Learning

### 1. Inventory System

Products have stock.

Example:

```
add-product
update-stock
view-stock
```

Concepts learned:

* state management
* encapsulation

---

### 2. Cart System

Instead of storing just product IDs:

```
Cart
 └── CartItem
        ├ product
        └ quantity
```

This teaches **object composition**.

---

### 3. Discount Strategy

Implement multiple discount types.

```
Discount
 ├── PercentageDiscount
 └── FixedDiscount
```

This introduces **polymorphism**.

---

### 4. Payment Simulation

```
Payment
 ├── CardPayment
 └── CashPayment
```

More inheritance practice.

---

### 5. Order History

```
Order
 ├ items
 ├ total
 ├ date
 └ status
```

This introduces **data modeling similar to real backend systems**.

# Important Warning ⚠️

Avoid adding **too many features** such as:

* search engine
* recommendation system
* payment gateways
* shipping simulation

Those will **explode the complexity**.
